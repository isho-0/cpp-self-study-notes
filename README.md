## 🌊 Commit Convention (커밋 규칙)

| 유형 | 설명 |
|------|------|
| `feat` | 새로운 문제 풀이 추가 |
| `docs` | 노트 정리 또는 문서 수정 |
| `refactor` | 기존 코드 개선 |
| `fix` | 버그 수정 또는 로직 수정 |
| `chore` | 설정 파일, 빌드 설정 등 변경 |

**예시**:

```bash
git commit -m "feat: week02 problem01 풀이 완료"
git commit -m "docs: week03 수업 노트 정리"
```
